# Investigation Report — Nmap Port Scan
**Date:** 2026-06-18  
**Analyst:** Abhinav Vijayvergia  
**Severity:** Low  

## Summary
A network reconnaissance scan was performed against Windows host 
ECO (192.168.1.xxx) from Kali Linux attacker machine. Nmap identified 
9 open ports and attempted OS fingerprinting.

## Scan Details
- Tool: Nmap 7.98
- Scan Type: SYN Scan + Aggressive (-sS -A)
- Duration: 195 seconds
- Target: 192.168.1.xxx

## Open Ports Discovered
| Port | Service | Notes |
|------|---------|-------|
| 135 | Microsoft RPC | Common lateral movement target |
| 139 | NetBIOS | Legacy Windows file sharing |
| 445 | SMB | High risk — historically exploited (EternalBlue) |
| 902 | VMware Auth Daemon | VMware installation exposed |
| 3306 | MySQL | Responding but unauthorized |
| 4343 | Unknown SSL | Unidentified service |
| 4449 | Unknown SSL | Unidentified service |
| 8089 | Splunkd HTTPS | SIEM exposed — high value target |
| 27000 | FlexLM | License manager |

## Second Scan (2026-06-28)
- Tool: Nmap 7.98
- Scan Type: SYN Scan only (-sS)
- Duration: 5 seconds
- Result: 7 open ports

| Port | Service | Notes |
|------|---------|-------|
| 135 | Microsoft RPC | Persistent across both scans |
| 139 | NetBIOS | Persistent across both scans |
| 445 | SMB | Persistent across both scans |
| 912 | VMware Apex Mesh | VMware service |
| 8000 | HTTP (Splunk Web UI) | Splunk dashboard exposed |
| 8080 | HTTP Proxy | Additional web port |
| 27000 | FlexLM | Persistent across both scans |

Note: Ports 3306, 4343, 4449, 8089 visible in Scan 1 but not Scan 2.
Likely due to -A flag doing deeper probing in Scan 1.

## OS Fingerprinting Result
- Nmap guess: Windows XP / Windows 7 / Windows Server 2012
- Actual OS: Windows 11 Home
- Conclusion: OS fingerprinting unreliable without open+closed port pair

## Splunk Detection
- EventCode 4688 (Process Creation): No events generated
- EventCode 4625 (Failed Logon): No events generated
- Conclusion: Nmap SYN scan is invisible to default Windows Event Logging

## Indicators of Compromise (IOCs)
- Source IP: Kali Linux (192.168.127.x)
- Target IP: 192.168.1.xxx
- Tool: Nmap 7.98
- Scan 1 Duration: 195 seconds (-sS -A)
- Scan 2 Duration: 5 seconds (-sS only)

## Lab Limitations & Findings
- Network reconnaissance generates zero Windows Event Log entries
- Default Windows logging has no visibility into port scanning activity
- Windows Firewall logs would capture this — not enabled in this lab
- Sysmon with network connection logging would partially detect this

## Recommendation
- Enable Windows Firewall logging to capture inbound connection attempts
- Deploy Sysmon for network-level process connection monitoring
- Monitor for sequential port connection attempts as scan signature
